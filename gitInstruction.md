# Инструкция по Git
Git-это утилита которая берет на себя вопросы контроля версий проекта.
## Команды для Git.
1. git status - получить информацию от Git o его текущем состоянии
2. git init - инициализация локального репозитория(то есть создав папку в этой папке с помощью этой команды создается папка __.git__, которая в дальнейшем и сохраняет все комиты всех файлов этой папки)
3. git add имя файла - добавляет файл к следующему комиту (без этой команды невозможно сохранение любых комитов)
4. git add . - добавляет все файлы, находящиеся в папке с репозиторием, в комит 
5. git commit -m "комментарий" - создание комита с сообщением о том что изменилось после последнего комита
6. git log - вывод на экран истории всех комитов с их хеш-кодами
7. git checkout 4t65 - переход от одного комита к другому, чтобы либо остаться и продолжить измнения в старом комите либо переделать его и перейти к следующему (где 4t65 - это первые четыре цифры уникального кода комита
8. git diff - увидеть разницу между текущим измненением файла, который еще не закомичен(сохранен), и комитом последней версии файла
9. git log --oneline - показать все комиты в сокращенном варианте(без имени администратора и без даты в одну строчку)
10. git log --oneline --all - показать все комиты всех файлов в сокращенном варианте
11. git branch - показывает все ветки которые есть( * обознаается в какой ветке мы находимся)
12. git checkout имя ветки - позволяет переходить по веткам 
13. git merge имя ветки - слияние: добавляется к текущей ветки, где мы находимся, та ветка, которая введена в команде
14. git branch -d имя ветки - удалить ветку, только ту которая уже слита(но не главную master)
15. git log --graph - позволяет видеть историю комитов в виде дерева
16. git log --name-only - позволяет видеть все комиты и какой комит к какому файлу относится
## Команды гит при работе с удалёнными репозиториями
1. git clone ссылка репозитория с Githab - добавляет репозиторий с Githab в локальный репозиторий (на наш компьютер)
2. cd имя папки, где находится репозиторий - изменить путь открытия папок. К примеру в одной папке находятся несколько папок с репозиториями чтобы перемещаться между ними.
3. cd .. - перемещаемся назад на одну папку или путь замкнутый в слэшах.
4. git push(толкать) - направляет все изменения(закомиченные) из локального репозитория в удаленный(на Githab)
5. git pull(стянуть) - отправляет все изменения(закомиченные) из удаленного репозитория в локальный
6. git push --set-upstream origin branch имя ветки - влить изменения после Fork(копирования удаленного репозитория взятого с аккаунта другого человека на Githabe) в удаленный репозиторий на нашем аккаунте в Githab из локального
## Некоторые тонкости пользования Git
* Удалять можно и не слитые ветки, только перед тем как удалить нужно перейти на другу ветку.

* Чтобы игнорировать файл в папке от репозитория нужно создать файл в папке, где репозиторий с именем *.gitignore.md* и прописать туда имя файла, который не нужен в репозитории.
